tableView的性能优化

By ivarspy at: 2017-08-06
  • 缓存行高
    1. 可以使用 NSCache | NSDictionary来缓存行高
  • 不要动态创建子视图
  • 使用shadowpath来设置阴影
  • 所有的子视图都预先创建
  • 如果不需要显示可以设置 hidden
  • 所有的子视图都应该添加到 contentView 上
  • 所有的子视图都必须指定背景颜色
  • 所有的颜色都不要使用 alpha
  • cell 栅格化
    异步绘制
    // 栅格化
    layer.shouldRasterize = true
    layer.rasterizationScale = UIScreen.mainScreen().scale
    // 异步绘制
    layer.drawsAsynchronously = true

补充: tableView的一些小技巧

  • 让UITableViewCell的分隔线宽度等于屏幕的宽度

    3种方法：

  • [ ] 自定义cell：

    1. 取消系统的分隔线
    2. 添加一个高度为1，宽度为屏幕宽度的UIView作为cell的子视图，通过设置frame或者约束使这个UIView始终在cell的底部
    3. 设置UIView的背景色，这样看起来就是cell的分隔线等于屏幕的宽度。

  • [ ] 通过设置系统的属性也能达到这个目的：（iOS8.0+）

    设置tableView的separatorInset和cell的layoutMargins:
    self.tableView.separatorInset = UIEdgeInsetsZero;
    cell.layoutMargins = UIEdgeInsetsZero;

  • [ ] 重写cell的setFrame方法（万能）：

  1. 取消系统的分隔线
  2. 重写cell的setFrame方法，减小cell的高度：
    - (void)setFrame:(CGRect)frame {
    `frame.size.height -= 5;`       //  height减小的值就是分隔线的高度
`[super setFrame:frame];`
    }
    3）、设置tableView的背景色就是分隔线的颜色
